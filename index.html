WPF-Validation by peterzhao

A light weight WPF validation framework is used to display validation results for each input field on WPF controls. How the validation results are generated totally rely on validation source, normally the View Models or Domain Objects, on which the views are bound to.

The built-in validation in WPF is very hard to use. Especially when your view has very complex validation logic between input fields. For example, there is an order input view. When the delivery method is set to pick up at store, there is no need to validation the delivery address input. But when the delivery method is set to deliver to home, delivery address input should be validated. There is a validation framework, ValidationScope, which provides powerful functionality to manage when to trigger a validation, how to display the validation results, where is validation source, and etc. However, the ValdiationScope framework is too heavy to use. Too much validation logic are scattered to the View side. The key point of MVVM pattern is to keep Views very straightforward and leave validation logic and hide/show logic to View Models. View Models should take care the complex logic of validation (enable/disable validation and validation rules) and Views just display the validation results according to the bindings to View Modes.
